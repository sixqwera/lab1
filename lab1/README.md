# Задание 1

## Задача 1

### Текст задачи

Сформировать список из вводимых строк.


### Алгоритм решения

Функция getlist (ввод данных):
Это рекурсивная функция для сбора строк. Она работает так: считывает строку с клавиатуры, и если строка не пустая, то «приклеивает» её к результату своего же следующего вызова. 
Рекурсия останавливается, когда пользователь вводит пустую строку (просто нажимает Enter) — в этот момент функция возвращает пустой список [], и вся цепочка слов собирается в один готовый список.
Функция main (управление):
Здесь запускается выполнение программы. Сначала выводится приглашение к вводу, затем результат работы функции getlist записывается в переменную myList. 
В конце программа выводит этот список на экран целиком, используя спецификатор %A, который сам рисует квадратные скобки и кавычки.

# Задание 2

## Задача 1

### Текст задачи

Найти суммарную длину введенных строк. (Признак конца ввода — пустая строка)

### Алгоритм решения

Функция readStrings (сбор списка):
Это рекурсивная функция для ввода данных. Она построчно считывает текст из консоли. 
Если строка не пустая, программа «приклеивает» её к результату своего следующего вызова (оператор ::). 
Как только пользователь вводит пустую строку, рекурсия останавливается и возвращает готовый список всех введённых слов.
Функция sumLength (подсчёт длины):
Здесь используется рекурсивный обход списка. Мы разбираем список на «голову» (первое слово) и «хвост» (все остальные слова). 
Функция берет длину первого слова (head.Length) и прибавляет к ней результат вызова самой себя для остатка списка. 
Когда список становится пустым, функция возвращает 0, и все длины складываются в итоговую сумму.
Функция main (управление):
Это точка входа в программу. Сначала она запускает readStrings и сохраняет результат в переменную mylist. 
Затем передает этот список в sumLength, чтобы получить общее число символов в переменную total. В конце программа выводит на экран сам список (через %A) и итоговую сумму длин (через %d).

### Текст задачи

Опишите функции, осуществляющие основные математические операции над комплексными

готовый класс Complex запрещено.

### Алгоритм решения

Определение структуры (type MyComplex):
Для начала я создал собственный тип данных. Это позволило объединить две дробные переменные (вещественную часть Re и мнимую Im) в один объект. Теперь программа понимает комплексное число как единое целое, а не как набор отдельных цифр.
Реализация математических операций:
Я описал основные функции для работы с числами:
Сложение и вычитание: Просто суммируем или вычитаем части Re и Im по отдельности.
Умножение: Здесь используется формула раскрытия скобок 
, где учитывается, что 
 (поэтому в вещественной части стоит минус).
Деление: Чтобы избавиться от мнимости в знаменателе, я ввел промежуточную переменную denom (знаменатель). Также добавлена проверка: если он равен нулю, программа выводит ошибку, а не «падает».
